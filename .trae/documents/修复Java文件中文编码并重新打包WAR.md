## 修复中文编码并重新打包WAR文件计划

### 问题分析

1. 所有Java文件存在中文乱码问题（如注释中的乱码）
2. 项目使用传统Servlet+JSP架构
3. 依赖库已存在于`WEB-INF/lib`目录
4. 数据库配置连接到阿里云RDS MySQL

### 实施步骤

#### 1. 修复中文编码问题

* 使用正确的编码（UTF-8）重新编写所有Java文件中的中文内容

* 检查并修复每个文件中的中文注释和字符串

* 确保文件本身保存为UTF-8编码

#### 2. 编译Java代码

* 创建`classes`目录用于存放编译后的class文件

* 使用`javac`命令编译所有Java文件

* 确保包含所有依赖库在classpath中

#### 3. 重新打包WAR文件

* 确保所有编译后的class文件正确放置

* 确保所有JSP文件、配置文件和资源文件都包含在WAR中

* 使用`jar`命令打包成WAR文件

### 具体操作

1. **修复中文编码**：逐个读取并修复31个Java文件中的中文乱码
2. **编译命令**：`javac -d classes -cp "WebContent/WEB-INF/lib/*" src/**/*.java`
3. **打包命令**：`jar -cvf shopping.war -C WebContent/ .`

### 预期结果

* 所有Java文件中文显示正常

* 编译成功，无错误

* 生成完整的WAR文件，可以部署到阿里云RDS

* 备份原始文件，防止修复过程中出现问题

* 检查编译错误，确保所有依赖正确

* 验证WAR文件结构，确保所有必要文件都包含在内

# 重新编译和打包项目计划

## 1. 清理旧的编译文件
- 删除现有的编译输出目录 `build` 和 `classes`
- 确保编译环境干净，避免旧文件影响新编译结果

## 2. 编译所有Java源代码
- 使用 `javac` 命令编译 `src` 目录下的所有Java文件
- 依赖项：`WebContent/WEB-INF/lib` 目录下的所有JAR包
- 编译输出目录：`WebContent/WEB-INF/classes`
- 命令示例：
  ```
  javac -cp "WebContent\WEB-INF\lib\*" -d "WebContent\WEB-INF\classes" "src\com\shopping\**\*.java"
  ```

## 3. 复制配置文件
- 将 `src` 目录下的配置文件复制到 `WebContent/WEB-INF/classes` 目录
- 包括：`db.properties` 和 `com/shopping/util/mail.properties`

## 4. 重新打包WAR文件
- 使用 `jar` 命令将 `WebContent` 目录打包成WAR文件
- 输出文件：`shopping.war`
- 命令示例：
  ```
  jar -cvf shopping.war -C WebContent .
  ```

## 5. 验证结果
- 检查 `WebContent/WEB-INF/classes` 目录是否包含所有编译后的 `.class` 文件
- 检查生成的 `shopping.war` 文件大小和内容

## 预期结果
- 成功编译所有Java文件，无编译错误
- 生成包含最新代码和配置的WAR文件
- 可直接部署到Java Web服务器运行

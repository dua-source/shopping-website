# 修复加入购物车跳转错误

## 问题分析
点击加入购物车后跳转到购物车页面时出现HTTP 500错误，原因是类型转换失败：
- `cartList.jsp` 文件中错误地使用了不存在的 `CartItem` 类
- 实际传递的是 `List<ShoppingCart>` 类型的数据
- 项目中只有 `ShoppingCart` 类，没有 `CartItem` 类

## 解决方案
修改 `cartList.jsp` 文件，将所有 `CartItem` 替换为 `ShoppingCart`：
1. 导入正确的类：`<%@ page import="com.shopping.model.ShoppingCart" %>`
2. 修改类型转换：`List<ShoppingCart> cartItems = (List<ShoppingCart>) request.getAttribute("cartItems");`
3. 修改循环中的类型：`for (ShoppingCart item : cartItems) {`

## 预期结果
- 购物车页面能够正常显示
- 加入购物车后跳转不再出现错误
- 购物车中的商品信息能够正确显示

## 执行步骤
1. 编辑 `WebContent/jsp/cartList.jsp` 文件
2. 将所有 `CartItem` 替换为 `ShoppingCart`
3. 重新编译项目
4. 重新打包WAR文件
5. 部署并测试

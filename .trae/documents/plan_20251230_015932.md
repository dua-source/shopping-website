# 修复查看订单详情跳转错误

## 问题分析
点击查看订单详情时出现HTTP 500错误，错误信息：
- `javax.servlet.ServletException: java.lang.ClassCastException: class org.apache.taglibs.standard.tag.common.core.ForEachSupport cannot be cast to class org.apache.jasper.runtime.TagHandlerPool`
- 原因：JSTL标签库版本与Tomcat版本不兼容，导致<c:forEach>标签无法正常工作

## 解决方案
将orderDetail.jsp中的JSTL <c:forEach>标签替换为传统的JSP脚本语法，避免JSTL库的版本兼容性问题。

## 修复步骤
1. 编辑`WebContent/jsp/orderDetail.jsp`文件
2. 将JSTL的<c:forEach>标签替换为传统的JSP for循环
3. 修改订单项的遍历方式，使用Java代码获取orderItems并遍历
4. 重新编译项目
5. 重新打包WAR文件
6. 部署并测试

## 预期结果
- 订单详情页面能够正常显示
- 订单项能够正确遍历和显示
- 不再出现500错误

## 修复示例
将：
```jsp
<c:forEach var="item" items="${order.orderItems}">
    <tr>
        <td>${item.productName}</td>
        <td>¥${item.unitPrice}</td>
        <td>${item.quantity}</td>
        <td>¥${item.subtotal}</td>
    </tr>
</c:forEach>
```

替换为：
```jsp
<% 
    List<OrderItem> orderItems = order.getOrderItems();
    for (OrderItem item : orderItems) {
%>
    <tr>
        <td><%= item.getProductName() %></td>
        <td>¥<%= item.getUnitPrice() %></td>
        <td><%= item.getQuantity() %></td>
        <td>¥<%= item.getSubtotal() %></td>
    </tr>
<% } %>
```

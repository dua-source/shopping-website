# 修复404错误计划

## 问题分析
index.jsp和其他JSP文件中的链接使用了绝对路径（以/开头），例如：
```jsp
<li><a href="/productList">首页</a></li>
```

当应用部署到Tomcat时，上下文路径是`/shopping-new`，所以浏览器会请求：
```
http://localhost:8080/productList
```
而不是正确的：
```
http://localhost:8080/shopping-new/productList
```

## 解决方案
修改所有JSP文件中的链接，使用`request.getContextPath()`获取上下文路径，或者使用相对路径。

## 修复步骤
1. 修改`index.jsp`文件中的链接，添加`${pageContext.request.contextPath}`前缀
2. 检查并修改其他JSP文件中的链接
3. 重新编译并打包WAR文件
4. 重新部署应用

## 具体修改示例
将：
```jsp
<li><a href="/productList">首页</a></li>
```
修改为：
```jsp
<li><a href="${pageContext.request.contextPath}/productList">首页</a></li>
```

或者使用相对路径：
```jsp
<li><a href="productList">首页</a></li>
```

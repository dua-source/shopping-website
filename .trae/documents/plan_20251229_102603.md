# 重新编译和打包项目计划

## 目标
- 重新编译项目的Java源代码
- 将编译后的类文件输出到正确目录
- 重新打包WebContent目录为WAR文件

## 实施方案

### 1. 编译Java源代码

**编译命令**：
```
javac -d WebContent/WEB-INF/classes -cp "WebContent/WEB-INF/lib/*" src/com/shopping/controller/*.java src/com/shopping/dao/*.java src/com/shopping/dao/impl/*.java src/com/shopping/model/*.java src/com/shopping/service/*.java src/com/shopping/service/impl/*.java src/com/shopping/util/*.java
```

**命令说明**：
- `-d WebContent/WEB-INF/classes`：指定编译后的类文件输出目录
- `-cp "WebContent/WEB-INF/lib/*"`：设置类路径，包含所有第三方库
- `src/.../*.java`：指定要编译的所有Java源文件

### 2. 打包WAR文件

**打包命令**：
```
jar -cvf shopping.war -C WebContent .
```

**命令说明**：
- `-cvf shopping.war`：创建名为shopping.war的WAR文件
- `-C WebContent .`：进入WebContent目录并打包所有内容

### 3. 验证打包结果

- 检查WAR文件是否成功创建
- 验证WAR文件大小和内容

## 执行步骤

1. 执行编译命令，将Java源代码编译为类文件
2. 执行打包命令，生成WAR文件
3. 检查生成的WAR文件

## 预期结果

- 所有Java源文件成功编译
- 生成的类文件位于WebContent/WEB-INF/classes/目录
- 生成shopping.war文件
- WAR文件包含所有必要的Web资源和编译后的类文件

## 注意事项

- 确保JDK已正确安装并配置
- 确保jar命令可用
- 确保WebContent/WEB-INF/classes/目录存在
- 编译过程中可能会出现警告，这是正常现象

## 后续步骤

- 将生成的shopping.war文件部署到Tomcat服务器
- 启动Tomcat服务器
- 访问网站验证功能

## 替代方案

如果编译命令执行失败，可以尝试：
1. 检查Java环境配置
2. 检查类路径是否正确
3. 检查源文件是否存在语法错误
4. 检查WebContent/WEB-INF/classes/目录是否有写入权限

如果打包命令执行失败，可以尝试：
1. 检查jar命令是否可用
2. 检查WebContent目录是否存在
3. 检查是否有足够的磁盘空间

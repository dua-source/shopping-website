## 重新编译和打包项目计划

### 1. 编译Java源代码
- 使用javac命令编译所有Java文件
- 设置classpath包含所有依赖库（WebContent/WEB-INF/lib/*）
- 设置输出目录为WebContent/WEB-INF/classes
- 编译src目录下的所有Java文件

### 2. 复制配置文件
- 确保db.properties文件从src目录复制到WebContent/WEB-INF/classes目录
- 确保mail.properties文件从src/com/shopping/util目录复制到WebContent/WEB-INF/classes目录

### 3. 重新打包WAR文件
- 使用jar命令将WebContent目录打包为war文件
- 输出文件名为shopping.war
- 替换原有的war文件

### 4. 验证结果
- 检查编译过程是否有错误
- 确认类文件是否正确生成
- 确认war文件是否成功创建

### 详细命令
```bash
# 1. 编译Java源代码
javac -cp "WebContent/WEB-INF/lib/*" -d "WebContent/WEB-INF/classes" "src/com/shopping/**/*.java"

# 2. 复制配置文件
copy "src/db.properties" "WebContent/WEB-INF/classes/db.properties"
copy "src/com/shopping/util/mail.properties" "WebContent/WEB-INF/classes/mail.properties"

# 3. 重新打包WAR文件
cd WebContent
jar -cvf "../shopping.war" .
cd ..
```
